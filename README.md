# Cybersecurity-Portfolio3-AI-Security

> **Current Status:** Planning stage. This portfolio is intended as the umbrella repository for future P3 AI security projects, which will be added as work begins.

## Overview
This repository is intended to serve as the root portfolio for my Portfolio 3 (P3) projects, focused on the cybersecurity of AI systems and AI-enabled applications.

P3 is planned as the next stage of my broader cybersecurity portfolio and is expected to focus on areas such as:
- AI application threat modeling
- prompt injection and AI abuse case analysis
- secure RAG (retrieval-augmented generation) design review
- AI security assessment and review workflows

## Purpose
The purpose of this portfolio is to provide a structured space for future projects related to AI security and the security review of AI-enabled systems.

As this portfolio develops, it is intended to reflect growth in areas such as:
- identifying AI-specific attack surfaces
- analyzing trust boundaries and data flows in AI applications
- documenting threats, abuse cases, and mitigations
- applying cybersecurity principles to AI-enabled tools and services

## Portfolio Progression
This portfolio is planned as part of a broader cybersecurity learning path:

- **P1** focuses on cyber core foundations such as lab architecture, segmentation, telemetry, and incident investigation.
- **P2** focuses on governance, risk, policy, and related security documentation and assessment work.
- **P3** is planned to focus on AI security, including the security of AI-enabled applications, AI-specific attack paths, and structured AI security review work.

Together, these portfolios are intended to show progression from core cybersecurity fundamentals into more specialized security domains.

## Planned Portfolio Structure
This portfolio is planned to include the following repositories:

### P3-1-ai-app-threat-model
A planned project focused on threat modeling for an AI-enabled application, including architecture, assets, trust boundaries, threats, and mitigations.

### P3-2-prompt-injection-and-ai-abuse-cases
A planned project intended to organize AI-specific abuse cases, including prompt injection, data exposure attempts, and unsafe tool-use scenarios.

### P3-3-secure-rag-review
A planned project focused on reviewing the security of a retrieval-augmented generation (RAG) system, including ingestion, retrieval, access control, and output validation risks.

### P3-4-ai-security-assessment-pack
A planned project intended to contain reusable AI security assessment materials such as review checklists, findings templates, vendor review questions, and security assessment guidance.

## Planned Skills Focus
As this portfolio is built out, it is expected to support development in areas such as:
- threat modeling
- AI security analysis
- security documentation

- secure design review
- security assessment reporting

## Why AI Security
As organizations continue adopting AI-enabled tools and applications, the attack surface is expanding to include risks such as prompt injection, insecure output handling, retrieval abuse, data leakage, unsafe tool execution, and over-permissioned AI workflows.

This portfolio is planned as a way to study, organize, and eventually demonstrate practical cybersecurity work related to those emerging risks.
